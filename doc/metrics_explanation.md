# 基准测试指标说明

## 概述

在 VLM（视觉-语言模型）选择任务中，我们需要评估不同方法预测模型性能的能力。以下是各项评估指标的详细说明。

---

## 1. 排序相关性指标

### 1.1 Kendall's τ (Kendall's Tau)

**定义**: 衡量两个排序之间的相关性

**范围**: [-1, 1]
- τ = 1: 完全一致的排序
- τ = 0: 无相关性
- τ = -1: 完全相反的排序

**计算原理**:
```
τ = (一致对数 - 不一致对数) / 总对数
```

**解读**: 
- 这是模型选择任务中最重要的指标
- 高 τ 值表示方法能够正确识别哪些模型更好
- 论文中通常报告 τ 值

### 1.2 Spearman 相关系数

**定义**: 衡量两个变量秩次（排名）之间的单调关系

**范围**: [-1, 1]

**与 Kendall's τ 的区别**:
- Spearman 对异常值更敏感
- Kendall's τ 更稳健

### 1.3 Pearson 相关系数

**定义**: 衡量两个变量之间的线性关系

**范围**: [-1, 1]

**注意**: 
- 假设线性关系
- 对异常值敏感
- 在模型选择中，我们更关心排序而非线性关系

---

## 2. 选择准确率指标

### 2.1 Top-5 Recall (R5)

**定义**: 预测的前5个最佳模型中，有多少在真实前5中

**范围**: [0, 1]

**计算**:
```
R5 = |预测Top5 ∩ 真实Top5| / 5
```

**示例**:
- 如果真实最佳5个模型是 [A, B, C, D, E]
- 预测最佳5个模型是 [A, C, F, G, H]
- R5 = 2/5 = 0.4

**解读**:
- R5 = 0.2 表示平均能找到1个真实最佳模型
- R5 = 0.5 表示平均能找到2-3个

### 2.2 Top-1 Accuracy

**定义**: 预测的最佳模型的实际准确率

**范围**: [0, 1]（实际上是准确率数值）

**解读**: 如果方法预测模型 A 最佳，则报告模型 A 的实际准确率

### 2.3 Oracle Accuracy

**定义**: 所有候选模型中最佳模型的准确率

**解读**: 这是理论上能达到的最优选择的结果

---

## 3. VEGA 特有指标

### 3.1 节点相似度 (Node Similarity, s_n)

**定义**: 衡量视觉特征与文本嵌入的对齐程度

**计算过程**:
1. 对于每个样本，计算其特征与所有类别文本嵌入的余弦相似度
2. 使用温度参数缩放后，用 softmax 归一化
3. 取每个样本对应预测类别的概率值
4. 按类别加权平均

**范围**: [0, 1]，越高越好

**解读**:
- 高节点相似度表示图像特征与文本嵌入对齐良好
- 如果模型预测准确，节点相似度应该较高

### 3.2 边相似度 (Edge Similarity, s_e)

**定义**: 衡量视觉图和文本图结构的相似性

**计算过程**:
1. 文本图：节点=类别嵌入，边=余弦相似度
2. 视觉图：节点=类别特征分布，边=Bhattacharyya距离
3. 计算两个图边矩阵的 Pearson 相关系数
4. 归一化到 [0, 1]

**范围**: [0, 1]，越高越好

**解读**:
- 高边相似度表示两类别的视觉关系与文本关系一致
- 例如：如果"猫"和"狗"在文本空间中相似，它们在视觉空间中也应该相似

### 3.3 VEGA 总分

**公式**: `VEGA = s_n + s_e`

**范围**: [0, 2]

**解读**:
- VEGA 分数越高，表示模型在下游任务上表现越好
- 两个分量从不同角度评估模型

## 4. LogME 指标

### 4.1 LogME 分数

**定义**: 基于贝叶斯框架计算特征与标签的最大证据

**计算过程**:
1. 假设特征与标签满足线性关系
2. 通过证据近似推断计算边际似然
3. 取对数得到 LogME 分数

**特点**:
- 不需要训练
- 计算效率高
- 原本设计用于监督学习，这里使用伪标签

**解读**:
- LogME 分数越高，表示特征与任务越匹配
- 在 VLM 选择中使用伪标签，可能引入噪声

---

## 5. 如何判断结果好坏

### 5.1 参考基准

根据 LOVM 论文和 SWAB 论文的结果：

| 方法 | R5 | τ |
|------|-----|------|
| ImageNet Baseline | 0.44 | 0.27 |
| ModelGPT | 0.45 | 0.27 |
| SWAB | **0.50** | **0.32** |

### 5.2 结果分析要点

1. **Kendall's τ**:
   - τ > 0.3: 较好
   - τ > 0.5: 很好
   - τ < 0.1: 几乎随机

2. **Top-5 Recall**:
   - R5 > 0.4: 能找到约2个真实最佳模型
   - R5 > 0.6: 能找到约3个真实最佳模型

3. **Top-1 Accuracy vs Oracle**:
   - 差距越小越好
   - 如果 Top-1 接近 Oracle，说明选择正确

## 6. 常见问题

### Q1: 为什么 VEGA 和 LogME 结果都是 NaN？

可能原因：
1. 数据加载失败
2. 维度不匹配
3. 计算过程中出现错误

**解决方案**: 检查数据加载函数，确保特征维度匹配

### Q2: Kendall's τ 为负值？

负值表示方法预测的排序与真实排序相反，可能是：
1. 方法不适用于该数据集
2. 存在 bug
3. 数据量太少导致随机波动

### Q3: Top-5 Recall 始终为 0？

可能原因：
1. 候选模型太少（<5个）
2. 方法完全无法区分模型优劣
3. 评估代码错误
